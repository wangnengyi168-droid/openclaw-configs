## 附录 A：固件 3.x vs 4.0 差异对照

| 特性 | 3.x (山立13台) | 4.0 (西图) |
|------|----------------|------------|
| 登录方式 | 浏览器Web UI（JSON+MD5不兼容） | `/Action/login` JSON+MD5可用 |
| API Token | ❌ 不支持 | ✅ 支持，`/Action/call` 带Token |
| API返回格式 | `{"Data": {...}, "ErrMsg": ""}` | `{"results": {...}, "code": 0}` |
| 写操作 | 必须浏览器DOM操作 | API可写（edit/add/set） |
| 安全加密 | 无SPA | SPA + 动态SALT加密 |
| 远程接入 | 云平台+浏览器自动化 | API Token直连 |

### 4.0 SPA+动态SALT对自动化的影响
- 4.0固件引入SPA (Single Page Application) + 动态SALT加密
- 登录时先GET `/Action/login`获取SALT，然后`MD5(MD5(密码)+SALT)`
- 自动化脚本需先获取SALT再计算签名，不能直接用明文密码

## 附录 B：云防火墙突破方案

### 问题
- firewall.ikuai8.com 与 yun.ikuai8.com 使用不同Cookie域
- JWT cookie 不跨子域名共享

### 解决方案
1. **浏览器自动化穿透**：Playwright登录 yun.ikuai8.com → 获取Cookie → 导航到 firewall.ikuai8.com → 注入Cookie
2. **本地788端口config_download**：通过内网788端口获取防火墙配置规则（非事件日志）
3. **云远程通道**：yun.ikuai8.com → 设备管理 → 远程 → 新标签打开Web后台

### 详细流程
```python
# Playwright 流程
1. page.goto('https://yun.ikuai8.com')
2. 输入账号密码登录
3. 获取 cookies
4. context.add_cookies([{name, value, domain: '.ikuai8.com'}])
5. page.goto('https://firewall.ikuai8.com')
6. 验证登录状态 → 提取防火墙规则
```

## 附录 C：PPTP → L2TP 迁移流程

### 迁移背景
- PPTP 安全性低（MPPE加密可被破解）
- L2TP/IPSec 更安全，且支持更多并发用户

### 西图三厂迁移案例（已验证）
1. **总部配置**：L2TP服务端，地址池 10.1.0.0/24
2. **静态路由**：指向L2TP客户端IP（NM: 10.1.0.10, 皮椅部: 10.1.0.11）
3. **客户端配置**：远程路由器L2TP客户端需**关NAT/开路由转发**
4. **验证**：三厂互通，PPTP逐步退役

### 关键教训
- 远程路由器L2TP客户端必须关NAT/开路由转发，否则总部无法访问其LAN
- 总部静态路由必须指向正确的L2TP客户端IP
- L2TP与PPTP可共存（不同地址池），现有用户可逐步迁移
