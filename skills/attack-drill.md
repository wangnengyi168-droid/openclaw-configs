# 攻防演练

## 快速开始

```bash
# 默认扫描当前网段 (192.168.101.0/24)
bash scripts/attack_drill_v2.sh

# 指定网段
bash scripts/attack_drill_v2.sh 192.168.2.0/24
bash scripts/attack_drill_v2.sh 192.168.110.0/24
```

## 山立集团专项攻防演练

> 基于 2026-05-01 全网审计定制的实战方案

### 目标网络
SL总部(东莞,61.145.210.85) → SD-WAN → 5家分厂（德安/博白/灵山/济源/K金）

### 5条红队攻击路径（按风险排序）

| # | 路径 | 攻击向量 | 耗时 | 风险 |
|---|------|----------|------|------|
| 1 | RDP暴破 | 8090端口→弱密码→跳板→iDRAC横扫 | 2h | 🔴 极高 |
| 2 | Web管理入侵 | 81端口→网关接管→全厂控制 | 1h | 🔴 极高 |
| 3 | SSH暴破 | 22端口→服务器→NAS数据窃取 | 1.5h | 🟠 高 |
| 4 | WiFi渗透 | 翻墙AP→内网直连→ARP欺骗 | 1h | 🟡 中高 |
| 5 | 钓鱼邮件 | 供应商伪装→端点→凭据→iDRAC | 1-3天 | 🔴 极高 |

### 蓝队5道防线
1. 公网边界收缩（关RDP/Web/SSH端口映射）
2. 网关+云防火墙（IPS/Web防护/Flood/威胁情报）
3. 内网检测（ARP全绑定/异常排查/VLAN隔离）
4. 端点加固（EDR/防火墙/WinRM限制/独立密码）
5. 日志告警（巡检异常检测/防火墙日志推送）

### 演练规则
- 时间窗口：2026-05-09~10（节后周末）
- 禁区：生产无盘服务器、群晖NAS、SQL数据库
- 紧急停止：业务中断立即终止

## 演练流程（6 阶段）

| 阶段 | 内容 | 耗时 |
|------|------|------|
| 1. 存活发现 | ARP 扫描，发现在线主机 | ~2s |
| 2. 端口扫描 | TCP 连接扫描 18 个高危端口 | ~30s |
| 3. 服务识别 | 识别服务版本和类型 | ~30s |
| 4. 漏洞检测 | NSE 脚本扫描已知漏洞 | ~60s |
| 5. 风险评分 | 0-100 安全评分 + 分级 | 即时 |
| 6. 报告生成 | Markdown 结构化报告 | 即时 |

## 高危端口列表

21(FTP), 22(SSH), 23(Telnet), 25(SMTP), 53(DNS), 80(HTTP), 110(POP3), 135(RPC), 139(NetBIOS), 143(IMAP), 443(HTTPS), 445(SMB), 1433(SQL Server), 3306(MySQL), 3389(RDP), 5432(PostgreSQL), 5900(VNC), 8080(HTTP代理), 8443(HTTPS)

## 风险评分规则

| 端口 | 权重 | 风险等级 |
|------|------|----------|
| RDP(3389) | ×15 | 🔴 严重 |
| SMB(445) | ×12 | 🔴 严重 |
| SQL(1433) | ×10 | 🔴 严重 |
| MySQL(3306) | ×8 | 🟠 高 |
| Telnet(23) | ×8 | 🟠 高 |
| FTP(21) | ×6 | 🟠 高 |
| HTTP(80) | ×3 | 🟡 中 |
| SSH(22) | ×2 | 🟢 低 |

**风险等级**：≥70 🔴高风险，≥40 🟡中风险，<40 🟢低风险

## 报告输出

- **位置**: `logs/attack_drill/攻防演练报告_YYYY-MM-DD_timestamp.md`
- **原始数据**: `logs/attack_drill/` 下各阶段文件
- **桌面副本**: 无（需要时手动复制）

## 注意事项

1. **无需 root 权限** — 使用 TCP Connect 扫描（-sT）
2. **超时控制** — 每主机 30s 端口扫描 + 60s 服务识别
3. **普通用户可用** — 不需要 sudo
4. **网段参数** — 第1个参数指定目标，不传则默认 192.168.101.0/24
